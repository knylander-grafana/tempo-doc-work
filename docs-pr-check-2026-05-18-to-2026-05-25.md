## Gap summary

1. **PR #6866: Document TraceQL metrics math.**
   - User impact: New query syntax for binary arithmetic between TraceQL metrics query results.
   - Recommended docs: Update `docs/sources/tempo/metrics-from-traces/metrics-queries/functions.md` and the functions summary in `docs/sources/tempo/metrics-from-traces/metrics-queries/_index.md`.
   - Include examples such as error-rate ratio:
     ```traceql
     ({ span:status = error } | rate()) / ({} | rate())
     ```
   - Cover supported operators (`+`, `-`, `*`, `/`), the requirement to wrap subqueries in parentheses, and current limitations noted in the PR, such as no scalar math like `1000 * ({} | rate())` yet.

2. **PR #7179: Update faster read path documentation for the new default.**
   - User impact: The docs currently instruct users to opt in, but span-only fetch is now enabled by default for eligible TraceQL metrics queries.
   - Recommended docs: Update `docs/sources/tempo/metrics-from-traces/metrics-queries/_index.md`.
   - Replace opt-in language with default-on behavior, document opt-out using `metrics_spanonly_fetch: false` and `with(spanonly_fetch=false)`, and clarify whether the "experimental" admonition still applies.

3. **PR #7184: Update backend-work dashboard documentation.**
   - User impact: Operators using the Tempo mixin get new redaction visibility and additional job panels, but the dashboard docs are incomplete.
   - Recommended docs: Update `docs/sources/tempo/reference-tempo-architecture/components/compaction.md` in the Monitoring section.
   - Add the Redaction row and the Dropped and Job Duration panels to the list of what the Tempo - Backend Work dashboard covers.

## Uncertain / needs engineering confirmation

- For #7179, confirm whether the faster span-only fetch path remains experimental after becoming the default. The existing docs use an experimental admonition, but the PR only changes default behavior and opt-out semantics.
